# Manager Personas

The Q&A manager has two layers:

- Fixed behavior: kind, concise, busy, no handholding.
- Dynamic persona: role, expertise, direct knowledge, hedged knowledge, blind spots, and answer style.

Use the manager whose knowledge naturally fits the scenario. Do not make every persona a generic "senior manager."

Current MVP recommendation:

- `forkly_order_history_export`: product-leaning team engineering manager.
- `forkly_empty_cart_bug`: tech-lead or platform-leaning engineering manager.

These personas are source material for scenario configs. The app currently reads the concrete persona embedded inside each scenario JSON file.
